Engineering Highlights

Features & Architecture

Examine the core technical implementation details that drive this analytics workspace.

Business Rule Filtering

Robust data selection logic enforcing strict thresholds for ratings, installs, reviews, and naming parameters before visual output.

Interactive Plotly Charts

All visualizations feature fully hoverable tooltips, legends, auto-resizing, and camera controls via Plotly JS rendering.

Revenue Analysis

Sophisticated business calculations modeling revenue as App Price × Installs, enabling paid application monetization analysis.

Sentiment Subjectivity

Calculates and merges average user review sentiment subjectivity with Play Store metadata to discover correlations with ratings.

Time-based Rendering

Visualizations are protected by timezone-aware scheduling, displaying charts only during specific hours of the day (IST).

Data Preprocessing

Robust regex patterns parsing ratings, formatting sizes into numerical MB values, and correcting date/version structures.

Case Study

About the Project

This project performs an end-to-end data analytics and business intelligence study of the Android mobile application marketplace. By merging application metadata with aggregated user reviews, we extract high-value insights regarding rating dynamics, pricing models, and user sentiments.

SaaS Business Rules

Every analysis applies strict multi-stage constraints mimicking corporate database filters (size limits, text lengths, content tiers).

Dynamic Date/Time Gating

Visualizations are contextually scheduled using timezone-aware logic to only render during optimal hours of the business day.

Dataset & Architecture

Raw Data Source: 10,841 Apps & 64,295 Reviews
Pre-processing: Sizes parsed to MB, dates converted to timestamps, text stripped of numbers.
Sentiment Integration: Polarity and subjectivity metrics aggregated and merged from review datasets.
Dashboard Platform: Vercel-hosted vanilla web application with embedded Plotly HTML engines.
